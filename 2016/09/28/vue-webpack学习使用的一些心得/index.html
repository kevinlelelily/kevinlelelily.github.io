vue+webpack学习使用的一些心得--浅谈，浅谈，浅谈

最近折腾vue+webpack，因为之前用angular1.3，写界面和controller比较费劲，angular总体来说是重了一些，同时比较看中webpack打包和管理机制。恰巧，公司web端聊天项目需要重构（之前用jquery和bootstrap），需要学习的只是还不少：vue语法自不必说，还有vue+vuex项目构架、webpack配置、es6的一些语法。但是一想到可以学习新东西啊，我那个小心脏就激动不已。他山之石，可以攻玉。

vue

  • vue官网给出了一套模板引擎，包管理运用了npm，这就相当方便了，并且本地可以起一个基于node的服务器，支持热插拔，代码写完保存后服务器端直接刷新，不用频繁刷浏览器了。整个vuedemo整合到本地配置完不到10分钟；
  • 关于vuex方面，自己也是看了好久vuex的文档，虽说是参考了redux的一些思想，但是没搞过react，所以一直把vuex当做数据存储模块用。三个方面：state,mutation,action。
    1. state里面变量的值在组件里是不能通过组件方法修改的，只能通过mutation方法进行修改，因为vuex本来就是状态管理的，如果在组件方法里就将state变量修改了，vuex是无法保存变量状态的，那么通过getter方法获得变量时的状态值就不确定了；
    2. mutation里面的方法不能套用其他mutation方法，也就是不支持mutation嵌套，但是可以处理一些业务逻辑；
    3. action是支持嵌套的，action方法必须是const，采用es6的export导出方法;action里面的方法名不要和组件里的方法名重名，否则会把组件里的方法名覆盖掉；

##webpack

  • webpack的配置文件网上有很多模板，这里主要记录下proxy server（代理服务器）这块,在devServer里配置
    proxy: {  
    '/cchelper-dfs/*': {  //engineer
        target: 'http://115.22.23.49:80',  
        secure: false  
    },
    '/http-bind/*': {
        target: 'http://115.29.113.27:7070',  
        secure: false  
     }
}
    这是我的在处理异步文件上传和登陆xmpp服务器时做的配置，配置完需要重新启动node服务器让配置文件生效，谨记！其他跟网上配置大同小异。
